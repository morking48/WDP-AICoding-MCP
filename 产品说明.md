# WDP AI Coding MCP 服务产品说明

## 一、什么是 MCP？

**MCP（Model Context Protocol）** 是 Anthropic 推出的开放协议，定义了 AI 助手与外部工具/数据源之间的标准通信方式。

简单来说：**MCP 让 AI 能够"调用外部能力"**，就像给 AI 装上了"手脚"，可以：
- 查询知识库
- 执行工具函数
- 读写文件
- 与外部系统交互

**类比理解**：
- 如果没有 MCP，AI 只能"聊天"，知识截止于训练数据
- 有了 MCP，AI 可以"动手"，实时获取最新知识、执行具体操作

---

## 二、什么是 Skill？

**Skill（技能）** 是 WDP 知识库中的**最小功能单元**，每个 Skill 对应一个特定的开发任务或功能点。

**Skill 的组成**：
- `SKILL.md` - 技能说明文档，定义功能描述、输入输出、使用示例
- `official-xxx.md` - 官方真值文档，包含准确的 API 参数和方法名
- `template/` - 代码模板（可选）

**Skill 的分类**：
- **BIM 相关**：构件查询、楼层管理、模型高亮等
- **GIS 相关**：地图加载、图层控制、要素编辑等
- **场景交互**：相机控制、对象操作、事件监听等
- **数据面板**：图表渲染、数据绑定、面板布局等

---

## 三、本 MCP 服务是用来做啥的？

本 MCP 服务是 **WDP（数字孪生平台）的 AI 辅助开发引擎**，核心功能是：

### 1. 智能工作流路由
用户输入需求 → 自动识别场景 → 匹配对应 Skill → 按步骤指导开发

### 2. 知识库查询
- 查询 WDP API 用法
- 获取官方文档真值
- 检索相关代码示例

### 3. 强制约束检查
编码前自动验证：
- ✅ 是否已读取必要 Skill
- ✅ 是否已查看官方文档
- ✅ 对象 Id 是否有效（防止用假值）
- ✅ 长任务是否启用状态管理

### 4. 上下文记忆
长对话自动保持状态，支持中断续接、多任务切换。

## 六、Remote Client 客户端介绍

`remote-client/` 是 **WDP MCP 的代理客户端**，采用"智能代理模式"设计。

### 核心特点

| 特性 | 说明 |
|------|------|
| **极简代码** | 仅 80 行，几乎不需要维护 |
| **永不更新** | 所有业务逻辑在服务器端，客户端只负责转发 |
| **自动同步** | 服务器更新后，客户端自动获取最新功能 |
| **缓存优化** | 工具定义缓存 60 秒，减少网络请求 |
| **多 IDE 支持** | 支持 Cline、Cursor、Windsurf、Claude Desktop 等 |

### 工作原理

```
┌─────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  IDE/AI助手  │────▶│  Remote Client   │────▶│   MCP 服务器    │
│  (用户界面)  │     │  (仅桥接转发)    │     │  (业务逻辑)     │
└─────────────┘     └─────────────────┘     └─────────────────┘
                           │                         │
                           │  1. 获取工具定义 (缓存)   │
                           │  2. 转发工具调用          │
```

**客户端职责**：
1. 从服务器获取工具定义（缓存 60 秒）
2. 将 IDE/AI 助手的工具调用转发到服务器
3. 返回服务器的结果给 IDE/AI 助手

**服务器职责**：
- 所有业务逻辑（知识库查询、工作流编排、约束检查）
- 工具定义管理
- 数据持久化

## 八、快速开始

### 1. 服务端部署（管理员）
```bash
cd mcp-knowledge-server
npm install
npm run build
npm start
```

### 2. 客户端配置（用户）
1. 解压 `remote-client.zip`
2. 在 IDE 中配置 MCP（参考部署指南）
3. 输入 Token，连接服务器
4. 测试：`调用start_wdp_workflow：了解WDP知识库机制`

### 3. 开始使用
```
调用start_wdp_workflow：实现BIM模型高亮功能

projectPath: D:/Projects/智慧园区
对象类别：BIM构件
对象 Id：暂无
```
