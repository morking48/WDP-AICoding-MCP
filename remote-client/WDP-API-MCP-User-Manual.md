# WDP MCP 用户操作手册

用法：
1. 先看下面“对象类别和对象 Id”
2. 再按场景复制对应模板

---

## 对象类别和对象 Id

### 常见对象类别

- `Hierarchy`：场景里的普通模型或层级实体
- `Path`：路径对象，用来画路线、轨迹、巡检路线
- `Poi`：点位标注
- `Window`：信息窗、弹窗、面板挂点
- `Particle`：粒子对象，比如烟雾、火焰、喷泉
- `Effects`：特效对象，偏视觉效果，不是普通模型
- `BIM构件`：BIM 单体、构件、楼层、房间、专业模型对象
- `GIS要素`：GIS 图层里的点线面要素

### 常见对象 Id

这些 Id 不是随便写的，必须来自项目数据、创建返回值、拾取结果、事件回调或查询结果。

- `eid`：实体 Id
  获取方式：通常来自创建返回值、实体查询结果、屏幕拾取结果
- `entityName`：实体名称或模型名称
  获取方式：通常来自场景已有模型名、创建时指定名称、点击回调返回值
- `customId`：项目侧自定义 Id
  获取方式：通常来自项目约定或创建对象时显式指定
- `seedId`：平台模型或 DaaS 上传模型编号
  获取方式：通常来自平台资源发布信息
- `nodeId`：BIM 构件、底板单体、实例节点 Id
  获取方式：通常来自 BIM 查询、BIM 点击回调、相关拾取结果
- `featureId`：GIS 要素 Id
  获取方式：通常来自 GIS 要素点击事件、要素属性查询结果

### 如果你现在没有 Id

复制模板：

```可复制指令
当前还没有准确对象 Id。

请先帮我判断应该通过什么方式获取：
1. 创建返回值
2. 屏幕拾取
3. 事件回调
4. 实体查询
5. BIM 构件查询
6. GIS 要素点击或查询

在没有确认获取方式前，不要直接假设 eid、nodeId、featureId 或其他对象标识。
```

## 场景 1：新任务启动

复制模板：

```可复制指令
请先通过WDP MCP服务读取下 wdp-entry-agent ，了解路由规则和调度机制。。

然后根据我后续的需求，自动判断应该继续读取哪些 skill。
如果这是长流程任务、跨多个能力域、需要跨多轮完成，需要使用 wdp-context-memory 做状态管理。

当前不需要写代码，告诉我你了解后本知识库的需求响应路径。
```

## 场景 2：需求大纲解析

适合：你有需求文档、需求文件夹、原型说明，想先产出结构化分析。

复制模板：

```可复制指令
请先按 wdp-entry-agent 和 wdp-intent-orchestrator 的流程处理这个需求。

需求文件路径：
【需求文件路径】下的【需求文件】

请输出《系统意图与架构设计报告》在需求文件路径下，并先给我确认，不要直接写代码。

要求：
1. 如果这是长流程任务或会跨多轮完成，继续使用 wdp-context-memory 做状态管理
2. 如果涉及对象操作，先明确对象类别，再明确对象 Id；如果没有 Id，就先说明该怎么获取
3. 如果涉及坐标、位置、范围、角度，必须列出具体参数
4. 如果涉及路径移动、相机跟随、天气切换、高亮、清理回收，要在报告里写出对应原生 API 能力
5. 如果信息不足，先列出缺失项，不要猜测补全
```

## 场景 3：单个需求描述解析

适合：你没有正式文档，只想把一段自然语言需求先整理成结构化任务说明。

复制模板：

```可复制指令
请先按 wdp-entry-agent 和 wdp-intent-orchestrator 的流程处理。

我现在只有一段需求描述，请先不要写代码，先整理成结构化实现说明，并告诉我还缺什么信息。

需求描述：
【把你的需求写在这里】

相关对象信息（如果不涉及，则去掉，你会跳过这部分的考虑）：
- 对象类别：【Hierarchy / Path / Poi / Window / Particle / Effects / BIM构件 / GIS要素 / 暂无】
- 对象 Id：【eid / entityName / customId / seedId / nodeId / featureId / 暂无】
- 坐标或位置参数：【如有则填写，没有就写“暂无”】

要求：
1. 如果这是长流程任务或跨多个能力域，继续使用 wdp-context-memory 做状态管理
2. 明确哪些信息已经足够，哪些信息必须补齐
3. 如果输入的对象 Id 或 类别 还没有或不清楚，请明确告诉我相关方法，来获取对应信息
4. 不要先进入编码
```

## 场景 4：在确认条件下启动编码

适合：需求报告已经确认，或者单个子任务边界已经明确，现在开始写代码。

复制模板：

```可复制指令
需求解析中存留的问题回复（无存留问题就跳过）：
问题1：
问题2：


请先按 wdp-entry-agent 的路由继续处理，并自动读取相关 skill。

如果这是长流程任务、会跨多轮、或者需要保留选中状态、相机状态、任务进度，继续使用 wdp-context-memory 做状态管理。

本轮唯一目标：
【一句话写清本轮只做什么】

相关工程路径：
【工程路径】

相关对象信息（如果不涉及，则去掉，你会跳过这部分的考虑）：
- 对象类别：【Hierarchy / Path / Poi / Window / Particle / Effects / BIM构件 / GIS要素 / 暂无】
- 对象 Id：【eid / entityName / customId / seedId / nodeId / featureId / 暂无】
- 坐标或位置参数：【如有则填写，没有就写“暂无”】

本轮要求：
1. 当前只处理这一个子任务，不要顺手扩展到别的模块
2. 写代码前必须读取对应的 official-*.md，禁止凭经验猜 API
3. 如果输入的对象 Id 或 类别 还没有或不清楚，请明确告诉我相关方法，来获取对应信息
4. 开始改动前，先告诉我：本轮目标、计划修改文件、读取了哪些 skill、依据的官方能力、验证方式
5. 如果发现需求范围变了，先暂停问我，不要继续扩写
```

## 场景 5：问题修复

适合：报错修复、行为异常、效果不对、调用没生效。

复制模板：

```可复制指令
请先按 wdp-entry-agent 的路由继续处理，并自动读取相关 skill 后帮我排查问题。

如果问题和历史上下文有关，继续使用 wdp-context-memory 续接状态。

当前问题：
【粘贴报错信息、现象描述、截图说明、复现步骤】

相关对象信息（如果不涉及，则去掉，你会跳过这部分的考虑）：
- 对象类别：【Hierarchy / Path / Poi / Window / Particle / Effects / BIM构件 / GIS要素 / 暂无】
- 对象 Id：【eid / entityName / customId / seedId / nodeId / featureId / 暂无】
- 坐标或位置参数：【如有则填写，没有就写“暂无”】

修复要求：
1. 先判断问题属于初始化、实体、相机、材质、BIM、GIS、覆盖物还是清理问题
2. 先查相关 official-*.md，再判断原因
3. 当前只修这个问题，不要顺手改别的模块
4. 如果输入的对象 Id 或 类别 还没有或不清楚，请明确告诉我相关方法，来获取对应信息
5. 开始修改前，先告诉我：问题判断、计划修改文件、读取了哪些 skill、验证方式
```

## 场景 6：当前进度记录

适合：长任务做了一半，要记录当前进度，方便下次续接。

复制模板：

```可复制指令
如果当前任务还没有启用状态管理，请先继续按 wdp-context-memory 规范整理当前进度。

当前任务目标：
【一句话目标】

请输出：
1. 已完成事项
2. 未完成事项
3. 当前关键对象信息
4. 当前关键文件
5. 当前风险点或待确认点
6. 下一轮最适合继续的单个子任务

不要顺手进入编码，只做进度整理和状态更新。
```

## 最短模板

### 最短版 A：新任务启动

```可复制指令
请先按 wdp-entry-agent 的规则处理我的后续需求；如果是长流程任务，继续使用 wdp-context-memory 做状态管理。当前先不要写代码，先告诉我执行路径。
```

### 最短版 B：先解析，不写代码

```可复制指令
请先按 wdp-entry-agent 和 wdp-intent-orchestrator 的流程处理。先不要写代码，先把下面需求整理成结构化实现说明，缺什么信息直接列出来，不要猜。
【需求描述】
```

### 最短版 C：开始编码

```可复制指令
请先按 wdp-entry-agent 的路由继续处理，并自动读取相关 skill 后开始编码。写代码前必须读取对应 official-*.md；如果是长流程任务，继续使用 wdp-context-memory 做状态管理。开始前先告诉我：本轮目标、计划修改文件、读取了哪些 skill、验证方式。
```
